# 用户订单列表与鞋类商品知识库设计

## 目标

为已登录的 Mall 会员提供“查看我的订单”能力，并初始化一份可被商品推荐子智能体检索的鞋类演示商品目录。实现保持当前 Deep Agent 主从协作风格，所有提示词和新增注释使用中文。

## 范围

本次包含：

- 在电商网关契约中增加当前用户订单列表查询。
- 在 Mall 网关中接入会员端 `/order/list`。
- 增加只读工具 `list_orders` 并注册到订单子智能体。
- 调整订单子智能体中文提示词，使其区分订单列表与单笔订单查询。
- 创建约 10 款演示鞋类商品的 Markdown 知识文档。
- 验证商品文档能被现有加载、切片和商品检索流程识别。

本次不包含：

- 修改 Mall 数据库或创建真实商城商品。
- 开放订单写操作。
- 将演示 SKU 用于下单或库存扣减。
- 改造 Mall 会员前台页面。

## 订单列表设计

### 网关契约

`EcommerceGateway` 增加异步方法：

```python
async def list_orders(
    self,
    user_id: str,
    status: int = -1,
    page_num: int = 1,
    page_size: int = 10,
) -> list[Order]:
    ...
```

`user_id` 必须来自 Agent 运行时上下文，不允许模型或浏览器直接指定。`page_size` 在适配器中限制为 1～20，避免一次读取过多数据。

### Mall 适配器

`MallEcommerceGateway.list_orders` 调用：

```text
GET {MALL_PORTAL_BASE_URL}/order/list
```

请求参数为 `status`、`pageNum`、`pageSize`。该接口使用当前会员 Bearer Token，因此结果天然限定在当前登录会员范围内。适配器将返回列表映射为统一 `Order` 模型，并使用运行时中的 `user_id` 填充领域对象。

展示订单标识时优先使用商城订单编号 `orderSn`，缺失时回退内部 `id`。列表项缺少商品明细时返回空商品列表，不为了补齐每笔详情而产生 N+1 请求。

### Agent 工具

新增只读工具 `list_orders(status=-1, limit=10)`：

- `status` 默认 `-1`，表示全部订单。
- `limit` 默认 10，限制为 1～20。
- 工具从 `ToolRuntime` 获取当前用户，不暴露 `user_id` 参数。
- 返回 `found`、`message`、`count` 和 `orders`。
- 空列表是正常结果，返回“当前账号暂无订单”。

订单子智能体拥有两个工具：

- 用户询问“我有哪些订单”“最近订单”“待发货订单”时调用 `list_orders`。
- 用户提供明确订单号并查询详情时调用 `get_order`。
- 不得根据对话内容伪造订单，也不得要求用户提供用户编号。

## 鞋类商品知识文档设计

新增文件：

```text
after_sales_agent/app/data/rag_sources/products/鞋类商品目录.md
```

文档包含约 10 款虚构演示商品，覆盖轻量跑鞋、缓震跑鞋、休闲板鞋、通勤鞋、篮球鞋、徒步鞋、雨天防水鞋、健步鞋等需求。每款商品使用独立二级标题，并包含：

- 商品名称与 SKU。
- 品类、演示价格、尺码、颜色和主要材质。
- 核心卖点、适用人群、适用场景。
- 脚型和尺码建议。
- 不适用场景或选购提醒。
- 检索关键词。

文档明确标注为“学习测试用演示数据”，避免与 Mall 真实在售库存混淆。商品推荐子智能体继续强制先调用 `search_products`，只根据检索结果推荐。

## 数据流

### 订单列表

```text
用户问题
  -> 主智能体委派 order_agent
  -> order_agent 调用 list_orders
  -> ToolRuntime 提供已认证 user_id 和 Mall 网关
  -> Mall 会员订单列表接口
  -> Order 领域模型列表
  -> 子智能体整理事实
  -> 主智能体生成中文答复
```

### 商品推荐

```text
鞋类商品 Markdown
  -> 知识库加载与切片
  -> 商品 Milvus Collection
  -> search_products
  -> product_agent 整理候选与匹配依据
  -> 主智能体生成推荐答复
```

## 错误处理与安全

- Mall 返回 401、403 时沿用现有认证和权限异常。
- Mall 网络或业务错误不伪装为空订单，继续抛出明确服务错误。
- 订单列表仅为只读工具，不触发取消、确认收货或售后申请。
- 订单归属以当前会员 Token 和运行时用户身份为准。
- 演示商品不声明真实库存，不承诺可以购买。

## 测试与验收

- 网关协议测试确认 `list_orders` 存在。
- Mall 网关测试确认请求路径、分页参数、状态筛选和字段映射。
- 工具测试确认默认限制、空列表和结构化输出。
- 子智能体测试确认同时注册 `list_orders` 与 `get_order`，并包含正确调用规则。
- RAG 加载测试确认鞋类文档分类为 `products`，且至少包含 10 个可检索商品段落。
- 运行 Ruff、JavaScript 语法检查和全量单元测试。

## 完成标准

- 用户说“我有哪些订单”时，订单子智能体具备真实查询当前账号订单列表的工具。
- 用户提供订单号时仍走原有单笔订单查询。
- 商品推荐只能从知识库中返回演示鞋类商品，不再声称系统没有商品数据。
- 新增代码保持简洁，中文提示词和中文注释清晰可读。
