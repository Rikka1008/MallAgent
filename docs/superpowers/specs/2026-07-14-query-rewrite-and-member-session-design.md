# Query 直接改写与会员会话状态设计

## 目标

先在主智能体和知识检索子智能体的中文提示词中实现直接 Query 改写，再补齐 Mall 会员退出登录、失效 Cookie 清理和前端会员状态展示。实现不增加额外 LLM 调用，不恢复旧用户名密码登录表单。

## Query 直接改写

### 适用范围

直接改写只用于商品知识和售后政策知识检索：

- 主智能体委派 `product_agent` 或 `policy_agent` 时，将必要的最近对话背景和当前问题整理成完整任务描述。
- `product_agent` 调用 `search_products` 前生成独立、明确、适合知识库检索的 Query。
- `policy_agent` 调用 `search_policy` 前执行相同处理。
- 订单号、SKU、金额、时间、售后单号等业务实体必须原样保留。

订单、物流、退款和售后写操作不执行 Query 改写，避免改变精确业务编号或用户明确意图。

### 改写规则

提示词要求子智能体：

1. 去除口语化赘词，将问题转换为简洁书面检索表达。
2. 根据任务中提供的最近对话消除“这个、那个、上次那双”等指代。
3. 修正常见错别字和不影响意图的大小写问题。
4. 保留用户核心诉求、限制条件和全部业务实体。
5. 不得增加对话中不存在的品牌、商品、政策或结论。
6. 指代无法唯一确定时先返回缺失信息，不调用检索工具猜测。
7. 改写结果只作为工具的 `query` 参数，不向用户展示内部改写过程。

示例：

```text
最近对话：用户正在比较“舒行宽楦健步鞋 SHOE-DEMO-008”。
当前问题：这个适合宽脚吗？
工具 Query：舒行宽楦健步鞋 SHOE-DEMO-008 是否适合宽脚用户
```

本次不实现独立 Query Rewriter、不增加额外模型调用，也不实现 HyDE。

## 会员会话状态

### 后端接口

新增：

```text
POST /api/auth/logout
```

接口删除 Agent 域下的 `mall_access_token` HttpOnly Cookie，并返回：

```json
{
  "authenticated": false,
  "user": null
}
```

删除 Cookie 时使用与创建时一致的 `path=/`、`secure`、`httponly` 和 `samesite=lax` 属性。

`GET /api/auth/status` 在以下情况返回匿名状态：

- 请求没有会员 Cookie。
- Mall 返回认证失效。

Mall 返回认证失效时，接口同时删除无效 Cookie，防止浏览器持续携带旧 Token。该接口不删除用户长期记忆，记忆继续按 Mall `user_id` 隔离。

### 前端状态

顶栏认证区域在已登录时显示：

- `Mall 已连接`。
- 当前会员用户名。
- `退出`按钮。

点击退出后：

1. `POST /api/auth/logout`，携带 `credentials: include`。
2. 成功后切换到匿名状态。
3. 禁用输入框和快捷操作。
4. 显示“前往 Mall 登录”，用户可由此切换账号。

退出请求失败时保留当前认证状态并显示可理解的错误，不假装已经退出。退出按钮在 checking、anonymous 和 error 状态隐藏。

## 安全边界

- 前端不能读取 HttpOnly Token，只能请求状态和退出接口。
- Query 改写不得编造知识或改变精确业务编号。
- 退出仅清除 Agent 会话 Cookie，不代替 Mall 门户自身的全局退出。
- 会员身份始终由 Mall `/sso/info` 验证，用户记忆不能决定登录身份。

## 测试

- 主智能体提示词包含委派时补齐上下文的规则。
- 商品和政策子智能体提示词包含完整直接改写规则和禁止编造约束。
- 订单、物流和退款子智能体不被要求改写精确编号。
- 退出接口返回匿名状态并发送过期 `mall_access_token` Cookie。
- 无效会员 Cookie 的状态检查同时发送清除 Cookie 响应。
- 前端包含会员用户名、退出按钮和 `/api/auth/logout` 请求。
- 退出成功后状态机进入 anonymous，输入区保持禁用。
- Ruff、JavaScript 语法检查和全量单元测试通过。

## 完成标准

- 模糊的商品或政策追问会在提示词约束下改写成独立工具 Query。
- 精确订单等业务标识不会被改写。
- 已登录会员可以退出，并通过 Mall 登录入口切换账号。
- 旧的 `test` Cookie 可以由页面退出操作清除，不依赖手工清理浏览器站点数据。
